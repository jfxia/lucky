Best fit

# On PR → review, test, deploy staging
workflow PRCheck

  parallel ReviewDiff, RunTests wait ->
  if passed then DeployStaging else PostFailure

Document Generation & Review

Great fit

Multi-stage document creation with dedicated writers, reviewers, and approvers. Audit trail for compliance.

# Draft → review ← iterate → finalize
workflow DocPipeline
  DraftDocument ->
  attempt
    ReviewDocument -> ApplyFeedback
  recover retry 3 -> approval
  FinalizeAndPublish

Security Auditing

Great fit

Multi-dimensional security review across dependency scan, static analysis, secret detection, and penetration testing — with explicit permission boundaries.

# Secure audit with least privilege
agent Scanner
  permissions allow filesystem.read deny shell.exec

workflow SecurityAudit
  parallel
    DependencyScan
    StaticAnalysis
    SecretDetection wait
  -> GenerateAuditReport

⛔ When Not to Use Lucky

❌ Number crunching — use Rust, Go, Python
❌ Real-time systems — LLM latency is unpredictable
❌ Web servers / APIs — Lucky is not a general-purpose language
❌ Single-shot LLM calls — just use a prompt
❌ Embedded devices — requires a runtime

⚡ What Makes Lucky Different

Compiled Orchestration

Lucky compiles to a portable DAG-based IR. The execution graph is verified at compile time — not at runtime.

First-Class Permissions

Every agent runs with explicit allow/deny permissions. Lexical inheritance, restrict-only semantics.

Declarative Recovery

Replace try/catch with attempt/recover: retry, fallback, escalate — all at language level.

Human in the Loop

approval blocks pause execution for human sign-off. Audit trail captures every decision.

Platform Neutral

Same .lir runs on Claude Code, Codex CLI, OpenCode, Cursor, Dify, WorkBuddy — via LTP.

Cost Budgets Built-in

--budget USD enforces cost limits. Every LLM call is tracked. Execution halts when budget is exceeded.
