# 技术交底书

## 一、发明名称

一种基于目标导向编排语言的AI智能体协同执行方法及系统

## 二、技术领域

本发明属于人工智能技术领域，具体涉及一种面向AI智能体（AI Agent）的声明式编排语言及其运行时系统，用于定义、管理和执行多个AI智能体的协同工作流。

## 三、背景技术

### 3.1 现有技术方案

目前，AI智能体的编排主要有以下技术方案：

**方案A：基于代码的SDK调用方式。** 通过编程语言调用LLM的API接口（如OpenAI SDK、Anthropic SDK），在代码中手动管理多个智能体的调用顺序、数据传递和错误处理。典型代表包括LangChain、LlamaIndex等框架。

**方案B：基于配置文件的流程编排。** 使用YAML或JSON配置文件定义智能体的执行顺序，由框架引擎解析配置并按序执行。典型代表包括Dify、Coze等工作流平台。

**方案C：基于自然语言的Agent对话。** 通过自然语言指令让一个主控Agent自行规划和调用其他Agent。典型代表包括AutoGPT、MetaGPT等。

### 3.2 现有技术的缺点

1. **SDK调用方式**存在以下不足：(a) 需要大量样板代码，开发者需手动处理状态管理、上下文传递、错误恢复等；(b) LLM调用被视为普通函数调用，缺乏对AI特性的语言级支持；(c) 多个Agent的并发控制和依赖管理完全依赖于开发者的编程能力。

2. **配置文件编排方式**存在以下不足：(a) 配置格式固定，缺乏表达复杂控制流的能力（如条件分支、循环、错误恢复）；(b) 无法对Agent的权限、记忆、推理策略等进行细粒度控制；(c) 缺乏编译时类型检查和优化。

3. **自然语言Agent方式**存在以下不足：(a) 执行过程非确定性，无法保证可重复性；(b) 缺乏显式的权限控制和审计能力；(c) 难以进行性能优化和成本控制。

### 3.3 本发明要解决的技术问题

1. 如何提供一种声明式的AI智能体编排语言，使开发者能够以接近自然语言的方式定义智能体的协同工作流，同时保持执行的确定性。

2. 如何在语言层面提供对AI特性的原生支持，包括模型声明、提示词模板、记忆管理、权限控制、人工审批等概念的语法级抽象。

3. 如何实现智能体间的上下文自动传播，消除手动参数传递的样板代码。

4. 如何将编排程序编译为目标无关的中间表示（IR），使其能够在不同的AI后端平台（如Claude Code、Codex CLI、OpenCode等）上一致执行。

## 四、发明内容

### 4.1 技术方案概述

本发明提出一种基于目标导向编排语言（Goal-Oriented Agent Language，简称GOAL）的AI智能体协同执行方法及系统。该系统包括：

**1. 声明式编排语言**：提供六层语义抽象——目标（Goal）、工作流（Workflow）、智能体（Agent）、任务（Task）、操作（Operation）、产物（Artifact），使开发者能够以声明式语法描述智能体协同方案。

**2. 编译器**：将编排语言源代码编译为目标无关的中间表示（IR），IR采用有向无环图（DAG）结构，图中的节点表示任务、工具调用、推理步骤、条件判断、审批节点等，边表示数据依赖和控制依赖。

**3. 运行时引擎**：包括基于优先级的DAG调度器、不可变分层上下文管理器、智能体记忆管理器、权限执行器和工具注册与执行器。

**4. 后端适配器**：通过标准化的Lucky工具协议（LTP）将IR映射到具体的AI平台执行。

### 4.2 核心技术特征

#### 4.2.1 六层语义抽象模型

本发明的编排语言定义了六个语义对象：

- **Goal（目标）**：定义"成功意味着什么"，声明成功标准而非实现细节
- **Workflow（工作流）**：定义"如何达成目标"，以DAG形式编排Agent和Task
- **Agent（智能体）**：定义"谁来执行工作"，拥有模型、记忆、工具、权限和提示词
- **Task（任务）**：最小的可调度、可检查点的工作单元，替代传统函数
- **Operation（操作）**：内置的语言原语（如search、summarize、review等）
- **Artifact（产物）**：每次执行产生的不可变输出

该六层模型将AI智能体编排的抽象层次从代码级别提升到语义级别。

#### 4.2.2 声明式工作流编排语法

工作流使用基于缩进的声明式语法，通过`->`箭头表示顺序依赖，同一缩进级别的节点默认并行执行：

```
workflow BuildAndDeploy
    Research
        ->
    Plan
        ->
    parallel
        Implement
        Test
    wait
        ->
    Deploy
```

该语法具备以下特性：(a) 顺序执行通过`->`显式声明；(b) 并行执行是默认行为，无需额外标注；(c) `parallel`/`wait`提供显式的并行屏障；(d) 支持条件分支（`if/else`）和错误恢复（`attempt/recover`）。

#### 4.2.3 AI模型作为语言原语

LLM（大语言模型）在本发明中被视为语言的基本组成部分，而非外部库调用：

```
model DeepSeek(
    provider = "deepseek",
    version = "deepseek-v4",
    temperature = 0.7,
)

use DeepSeek    -- 设置默认模型

agent Researcher
    use GPT     -- 为特定Agent覆盖模型
```

编译器能够理解模型声明、提示词结构和输出置信度等AI特有概念，并据此进行类型检查和优化（如模型路由、响应缓存）。

#### 4.2.4 上下文自动传播机制

本发明实现了上下文（Context）在工作流→Agent→Task层次结构中的自动传播，消除了手动参数传递：

```
workflow BuildFeature
    context
        user: String
        repo: URI
        feature_spec: String

    Designer.design()     -- 自动获得context.user等
        ->
    Coder.implement()     -- 自动获得context.user及Designer的输出
```

上下文采用不可变分层结构：每个作用域创建新的上下文层，子层可覆盖父层条目但不可修改。这种设计使得执行过程可检查点化和可重放。

#### 4.2.5 基于DAG的中间表示（IR）

编排语言源代码被编译为与后端无关的IR。IR采用有向无环图结构：

- **HIR（高层IR）**：对应Agent/Task粒度的执行DAG，包含Goal、Workflow、Agent、Task等18种节点类型
- **MIR（中层IR）**：SSA形式的控制流图，支持标准编译器优化（死代码消除、常量折叠、公共子表达式消除等）
- **LIR（低层IR）**：线性指令序列，可直接解释执行

同一IR可在不同后端平台（Claude Code、Codex CLI、OpenCode等）上一致执行，通过LTP协议进行通信。

#### 4.2.6 能力安全模型

每个Agent在执行时具有显式声明的权限集：

```
permissions
    allow
        filesystem.read
        git.clone
        browser.search
    deny
        filesystem.delete
        shell.exec
```

权限采用词法作用域继承：Agent从其所属的Workflow继承权限，并可进一步限制但不能扩展。权限检查支持通配符匹配。

#### 4.2.7 声明式错误恢复

本发明采用声明式的错误恢复策略替代传统的try/catch异常处理：

```
attempt
    deploy_to_production
recover
    retry 3 with backoff exponential(max: 5m)
recover
    fallback deploy_to_staging
recover
    human escalate "Deployment failed"
```

支持的恢复动作包括：重试（支持指数退避）、回退（执行替代任务）、人工升级、中止和跳过。

### 4.3 技术效果

1. **开发效率提升**：通过声明式语法和上下文自动传播，将多智能体协同的开发代码量减少一个数量级。

2. **执行确定性**：尽管AI推理是概率性的，但编排结构是确定性的——DAG的拓扑结构和执行顺序在编译时确定，可重复执行。

3. **后端可移植性**：通过IR抽象层和LTP协议，同一编排程序可在不同AI平台上执行，无需修改源代码。

4. **安全性增强**：通过显式权限控制和人工审批门，实现对AI智能体的最小权限原则管理。

5. **成本可优化**：运行时引擎支持模型路由（简单任务使用廉价模型）、响应缓存和批量处理，降低Token消耗。

## 六、具体实施方式

### 6.1 实施例1：多智能体代码审查系统

以下是一个使用本发明编排语言实现的多智能体代码审查系统：

```lucky
project CodeReviewBot

use DeepSeek

model DeepSeek(
    provider = "deepseek",
    version = "deepseek-v4",
    temperature = 0.2,
)

agent SecurityReviewer
    model DeepSeek
    tools Git, Filesystem
    permissions allow git.diff, allow git.comment
    prompt SecurityPrompt

agent StyleReviewer
    model DeepSeek
    tools Git, Filesystem
    permissions allow git.diff, allow git.comment

agent ReportAggregator
    model DeepSeek
    tools Git
    permissions allow git.comment

workflow FullReview
    FetchPR(pr_number = context.pr)
        ->
    parallel
        SecurityReviewer.review(diff = context.diff)
        StyleReviewer.review(diff = context.diff)
    wait
        ->
    ReportAggregator.merge(
        security = context.security_findings,
        style = context.style_findings,
    )
        ->
    PostComment(pr_number = context.pr)
```

该实施例展示了：(a)多Agent并行执行（安全审查和风格审查同时进行）；(b)上下文自动传播（PR差异数据自动传递给各Agent）；(c)Agent级别的权限控制（仅允许读取差异和发布评论）。

### 6.2 实施例2：带有错误恢复的部署工作流

```lucky
workflow ResilientDeploy
    BuildProject()
        ->
    RunTests()
        ->
    attempt
        DeployToProduction()
    recover
        retry 3 with backoff exponential(max: 10m)
    recover
        fallback DeployToStaging()
    recover
        human escalate "Production deployment failed"
```

该实施例展示了声明式错误恢复机制：先重试3次（指数退避），失败后回退到预发布环境，再次失败后升级给人工处理。

### 6.3 编译与执行流程

1. 开发者编写`.lk`源文件，定义Agent、Task、Workflow和Goal。

2. 编译器执行：词法分析→语法分析→语义分析→HIR生成→MIR降级→IR序列化。

3. 运行时引擎加载IR，构建执行DAG，通过优先级调度器执行节点。

4. 对于LLM调用节点，运行时通过后端适配器调用相应的AI服务（如DeepSeek API）。

5. 对于工具调用节点，运行时通过工具注册表执行相应的工具（如Git操作、文件系统读写）。

6. 运行时记录执行状态，支持检查点保存和故障恢复。

## 七、权利要求建议

1. 一种基于目标导向编排语言的AI智能体协同执行方法，其特征在于，包括以下步骤：
   (a) 接收以声明式编排语言编写的源程序，所述源程序包含目标(Goal)、工作流(Workflow)、智能体(Agent)和任务(Task)的声明；
   (b) 将所述源程序编译为有向无环图(DAG)形式的中间表示，其中节点表示执行单元，边表示依赖关系；
   (c) 通过运行时引擎执行所述DAG，其中同一缩进级别的无依赖节点并行执行，通过`->`箭头显式声明的节点串行执行；
   (d) 在执行过程中，上下文(Context)从工作流作用域自动向下传播至智能体和任务作用域。

2. 根据权利要求1所述的方法，其特征在于，所述编排语言将大语言模型(LLM)作为语言原语，通过`model`关键字声明模型配置，通过`use`关键字进行模型选择，编译器和运行时能够基于模型声明进行类型检查和执行优化。

3. 根据权利要求1所述的方法，其特征在于，所述编排语言提供声明式的错误恢复语法，支持`attempt/recover/retry/fallback/human`等恢复动作的组合链式声明。

4. 根据权利要求1所述的方法，其特征在于，所述中间表示分为三个层次：高层IR(HIR)对应Agent/Task粒度的执行DAG，中层IR(MIR)为SSA形式的控制流图，低层IR(LIR)为线性指令序列。

5. 根据权利要求1所述的方法，其特征在于，所述运行时引擎通过标准化的工具协议(LTP)与不同AI平台后端进行通信，使得同一IR可在多种AI平台上一致执行。

6. 一种基于目标导向编排语言的AI智能体协同执行系统，其特征在于，包括：
   (a) 编译器模块，用于将编排语言源程序编译为中间表示；
   (b) 运行时引擎模块，包括DAG调度器、上下文管理器、智能体记忆管理器、权限执行器和工具注册表；
   (c) 后端适配器模块，通过标准化协议将IR映射到具体AI平台执行。

7. 一种计算机可读存储介质，其上存储有计算机程序，该程序被处理器执行时实现权利要求1-5任一项所述的方法。
